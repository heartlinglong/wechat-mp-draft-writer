# Workflow Reference

## Fast checklist

1. 先选题，不急着写
2. 找 2-4 个高质量信息源交叉确认
3. 提炼一个核心判断句
4. 先想标题，再想开头两句钩子
5. 正文一句一段，尽量短
6. 结尾给方法，不要只是感慨
7. 生成 HTML
8. 需要时调用 `python3 /root/.openclaw/workspace/scripts/wechat_mp_draft.py`

## Title heuristics

好标题常见结构：
- 反差：大家以为 X，结果真正值钱的是 Y
- 悬念：最危险的不是功能，而是后面那只手
- 情绪张力：这次不是升级，是行业重新洗牌
- 利益相关：谁会先赚钱，谁会先失业，谁会先被替代

避免：
- 纯感叹号堆砌
- 空泛大词
- 一眼假大空的“震惊体”

## Ending heuristics

结尾最好回答这几个问题里的 1-2 个：
- 普通读者现在能拿它做什么？
- 创业者/从业者该盯哪 3 个后续信号？
- 什么迹象出现后，要立刻改判断？
- 如果现在只能做一个动作，最值得做什么？

## HTML note

Keep the HTML simple and UTF-8 encoded. Plain `<p>` blocks are usually enough.
